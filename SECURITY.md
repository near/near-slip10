# Security Policy

`near-slip10` performs SLIP-0010 key derivation, so vulnerabilities here can directly
affect the safety of users' keys. Please treat any potential issue as sensitive.

**Do NOT open a public GitHub issue for security vulnerabilities.**

To report a vulnerability, please email the NEAR Security Team at
[security@near.org](mailto:security@near.org), preferably with a proof of concept. You
may also submit reports through NEAR's bug bounty program on
[HackenProof](https://hackenproof.com/near/near-protocol).

We ask that you do not use other channels or contact project contributors directly, and
that you give us a reasonable amount of time to address the issue before any public
disclosure.

Non-vulnerability security suggestions (e.g. new hardening ideas) are welcome as regular
GitHub issues.
